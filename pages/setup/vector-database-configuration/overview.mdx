# Vector Databases

Your vector database is set system-wide and cannot be configured per workspace like LLMs can.

<Callout type="error" emoji="‼️">
  **Caution!**  
  Avoid switching (“hopping”) between vector databases. RealTimeX does not automatically port your already embedded data.  
  <br/><br/>
  To migrate, you must delete and re-embed every document in all workspaces for the new vector database.
</Callout>

RealTimeX supports multiple vector database providers out of the box.
